# Создание виртуальной машины

Создаю ВМ аналогичную ВМ из 1-го ДЗ.
# Установка Docker Engine

Обновляю пакеты:
```bash
sudo apt update && sudo apt upgrade
```

Удаляю пакеты, которые могут помешать установке docker:
```bash
sudo apt remove $(dpkg --get-selections docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)
```

Скачиваю скрипт установки docker:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
```

Запускаю скрипт установки:
```bash
sudo sh ./get-docker.sh
```

# PostgreSQL в Docker

Создаю директорию для персистентного хранения данных PostgreSQL в контейнере:
```bash
sudo mkdir /var/lib/postgres
```

Скачиваю образ контейнера с PostgreSQL:
```bash
sudo docker pull postgres:14-alpine3.22
```

Создаю docker сеть, чтобы сервер и клиент могли видеть друг друга:
```bash
sudo docker network create postgres
```

Запускаю контейнер `postgres-server`:
```bash
sudo docker run -d -e POSTGRES_PASSWORD=1234 -p 5432:5432 -v /var/lib/postgres:/var/lib/postgresql/data --rm --name postgres-server --net postgres postgres:14-alpine3.22
```

`-d` - запуск в режиме detached.
`-e POSTGRES_PASSWORD=1234` - задаю пароль пользователя postgres.
`-p 5432:5432` - пробрасываю порт из контейнера на ВМ.
`-v /var/lib/postgres:/var/lib/postgresql/data` - создаю персистентное хранилище данных контейнера.
`--rm` - автоматическое удаление контейнера после установки. Не нужно будет писать `docker rm postgres-server`
`--name postgres-server` - фиксированное имя контейнера. Без этого docker создаст случайное имя и контейнер придётся искать.
`--net postgres` - указываю в какой docker сети создать контейнер.
`postgres:14-alpine3.22` - название образа.

Запускаю контейнер `postgres-client`:
```bash
sudo docker run -it -e POSTGRES_PASSWORD=1234 --rm --name postgres-client --net postgres postgres:14-alpine3.22 bash
```

`-it bash` - запуск в интерактивном режиме.

Из `postgres-client` подключаюсь к `postgres-server`:
```bash
psql -h postgres-server -U postgres
```
Ввожу ранее заданный пароль.

Создаю таблицу с данными:
```sql
create table shipments(id serial, product_name text, quantity int, destination text);

insert into shipments(product_name, quantity, destination) values('bananas', 1000, 'Europe');
insert into shipments(product_name, quantity, destination) values('bananas', 1500, 'Asia');
insert into shipments(product_name, quantity, destination) values('bananas', 2000, 'Africa');
insert into shipments(product_name, quantity, destination) values('coffee', 500, 'USA');
insert into shipments(product_name, quantity, destination) values('coffee', 700, 'Canada');
```

# Перезапуск контейнера postgres-server

Останавливаю контейнер `postgres-server`:
```bash
sudo docker stop postgres-server
```
Он автоматически удаляется т.к. был запущен с ключом `--rm`

Заново запускаю контейнер `postgres-server`:
```bash
sudo docker run -d -e POSTGRES_PASSWORD=1234 -p 5432:5432 -v /var/lib/postgres:/var/lib/postgresql/data --rm --name postgres-server --net postgres postgres:14-alpine3.22
```

С домашнего компьютера проверяю, что данные на месте:

Вывод: данные не удалились т.к. были сохранены в ВМ, а не в контейнере.
