# Создание виртуальной машины

Создаю ВМ с ОС Ubuntu 26.04 в Virtual Box аналогичную ВМ из 1-го ДЗ, только с увеличенным до 50ГБ диском.

Устанавливаю гостевые дополнения:
```bash
sudo apt update && sudo apt upgrade
sudo apt install build-essential dkms linux-headers-$(uname -r)
sudo apt install virtualbox-guest-utils virtualbox-guest-x11
sudo reboot
```

Нажимаю `Devices` -> `Insert Guest Additions cd image`.
Перехожу в директорию диска:
```bash
cd /run/media/stanislav/VBox_GAs_7.2.8
```
Запускаю установку дополнений:
```bash
sudo ./VBoxLinuxAdditions.run
```
# Установка Docker Engine

Удаляю пакеты, которые могут помешать установке docker:
```bash
sudo apt remove $(dpkg --get-selections docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc | cut -f1)
```

Скачиваю скрипт установки docker:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
```

Запускаю скрипт установки:
```bash
sudo sh ./get-docker.sh
```

# План работ

Будет запущено три контейнера:
- pg-master - основной PostgreSQL на который будет приходить нагрузка.
- pg-replica - реплика основного PostgreSQL, с которой буду снимать бэкап.
- pg-restore - отдельный PostgreSQL на котором будет происходить восстановление.

# Подготовка к запуску контейнеров

Создаю директорию проекта:
```bash
sudo mkdir hw05
cd hw05
```

Настраиваю docker-compose:
```bash
sudo nano docker-compose.yml
```
```
services:
  pg-master:
    image: postgres:16
    container_name: pg-master
    user: postgres
    environment:
      POSTGRES_PASSWORD: 1234
      POSTGRES_DB: loyalty
    volumes:
      - ./master_init.sql:/docker-entrypoint-initdb.d/init.sql
      - ./walg.json:/etc/wal-g/walg.json
      - ./backup_storage:/backup
      - ./wal-g:/usr/local/bin/wal-g
      - ./master_pg_hba.conf:/etc/postgresql/pg_hba.conf
    command: >
      postgres
      -c wal_level=replica
      -c archive_mode=on
      -c archive_command='wal-g --config /etc/wal-g/walg.json wal-push "%p"'
      -c archive_timeout=60
      -c max_wal_senders=5
      -c wal_keep_size=1024
      -c hba_file='/etc/postgresql/pg_hba.conf'
    networks:
      - pgnet

  pg-replica:
    image: postgres:16
    container_name: pg-replica
    user: postgres
    environment:
      POSTGRES_PASSWORD: 1234
    volumes:
      - ./walg.json:/etc/wal-g/walg.json
      - ./backup_storage:/backup
      - ./wal-g:/usr/local/bin/wal-g
      - ./replica_entrypoint.sh:/entrypoint.sh
    entrypoint: /bin/bash /entrypoint.sh
    depends_on:
      - pg-master
    networks:
      - pgnet

networks:
  pgnet:
```

Скачиваю wal-g:
```bash
wget https://github.com/wal-g/wal-g/releases/download/v3.0.8/wal-g-pg-24.04-amd64
```

Меняю название бинарника:
```bash
sudo mv wal-g-pg-24.04-amd64 ./wal-g
```

Даю права на выполнение:
```bash
sudo chmod +x ./wal-g
```

Проверяю версию:
```bash
wal-g --version
```
![](../project/Pasted%20image%2020260531174640.png)

Настраиваю конфигурацию wal-g:
```bash
nano walg.json
```
```
{
  "WALG_FILE_PREFIX": "/backup",
  "WALG_COMPRESSION_METHOD": "brotli",
  "PGHOST": "/var/run/postgresql",
  "PGUSER": "postgres",
  "PGPASSWORD": "1234"
}
```

Настраиваю файл с правилам подключения к СУБД на мастере:
```bash
nano master_pg_hba.conf
```
```
local   all             all                                     trust
host    all             all             0.0.0.0/0               md5
host    replication     all             0.0.0.0/0               md5
```

Настраиваю скрипт инициализации таблицы:
```bash
nano master_init.sql
```
```SQL
CREATE TABLE customers (
    id SERIAL PRIMARY KEY,
    name TEXT,
    discount FLOAT
);
INSERT INTO customers (name, discount)
SELECT 'Customer ' || s, random()*30
FROM generate_series(1,100000) s;
```

Настраиваю скрипт инициализации реплики:
```bash
nano replica_entrypoint.sh
```
```bash
#!/bin/bash
until pg_isready -h pg-master -U postgres; do sleep 1; done

rm -rf /var/lib/postgresql/data/*

PGPASSWORD=1234 pg_basebackup -h pg-master -U postgres -D /var/lib/postgresql/data -P -R

echo "hot_standby = on" >> /var/lib/postgresql/data/postgresql.auto.conf

exec docker-entrypoint.sh postgres
```

Настраиваю права на запуск:
```bash
chmod +x replica_entrypoint.sh
```

Создаю директорию для бэкапов:
```bash
mkdir backup_storage
```

# Запуск контейнеров

Запускаю контейнеры:
```bash
sudo docker compose up -d
```

Проверяю корректность работы контейнеров:
```bash
sudo docker ps
```
![](../project/Pasted%20image%2020260531223517.png)

В случае ошибок, можно закрыть все контейнеры через:
```bash
sudo docker compose down -v
```

Проверяю, активна ли репликация:
```bash
sudo docker exec pg-master psql -U postgres -c "SELECT * FROM pg_stat_replication;"
```
![](../project/Pasted%20image%2020260531223556.png)

Нагружаю мастер:
```bash
sudo docker exec pg-master pgbench -U postgres -i loyalty
sudo docker exec -d pg-master pgbench -U postgres -c 10 -T 60 loyalty
```

Проверяю, что pg_bench выполняется:
![](../project/Pasted%20image%2020260531225034.png)

Снимаю бэкап с реплики:
```bash
sudo docker exec pg-replica wal-g --config /etc/wal-g/walg.json backup-push /var/lib/postgresql/data
```

Запускаю контейнер с PostgreSQL для восстановления:
```bash
sudo docker run -it --rm \
  --user postgres \
  -v $(pwd)/backup_storage:/backup \
  -v $(pwd)/wal-g:/usr/local/bin/wal-g \
  -v $(pwd)/walg.json:/etc/wal-g/walg.json \
  -v hw05_restore_data:/var/lib/postgresql/data \
  postgres:16 bash
```

Очищаю каталог данных:
```bash
rm -rf /var/lib/postgresql/data/*
```

Загружаю последний бэкап:
```bash
wal-g --config /etc/wal-g/walg.json backup-fetch /var/lib/postgresql/data LATEST
```

Настраиваю восстановление в postgresql.auto.conf:
```bash
echo "restore_command = 'wal-g --config /etc/wal-g/walg.json wal-fetch \"%f\" \"%p\"'" > /var/lib/postgresql/data/postgresql.auto.conf

echo "recovery_target_timeline = 'latest'" >> /var/lib/postgresql/data/postgresql.auto.conf

touch /var/lib/postgresql/data/recovery.signal
```

Запускаю postgreSQL:
```bash
pg_ctl start -D /var/lib/postgresql/data -l /tmp/pg.log
```

Проверяю количество строк:
```bash
psql -U postgres -d loyalty -c "SELECT count(*) FROM customers;"
```

Восстановление из бэкапа выполнено.
